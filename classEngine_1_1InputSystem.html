Detailed Description

Handles all input processing for the application.

Manages keyboard and mouse input, camera controls, and input event handling. Provides an interface between GLFW input and the game systems.

Processes registered key callbacks and manages built-in key actions, specifically toggling cursor lock when the ESC key is pressed.

◆ RegisterKeyCallback()

void Engine::InputSystem::RegisterKeyCallback ( int  key,
KeyCallback  callback 
)

Registers a custom callback function for a specific key event.

Parameters
keyThe GLFW key code for which the callback is being registered.
callbackA function to be called when the specified key is pressed or released.

This method allows dynamic registration of custom key event handlers. The callback will be stored in the internal key callbacks map and can be triggered during key events. Registering a new callback for an existing key will replace the previous callback.

Note
The callback is moved into the map to avoid unnecessary copying.

◆ UpdateCursorState()

void Engine::InputSystem::UpdateCursorState ( )
private

Updates cursor visibility and lock state.

Updates the cursor state in the window based on the current cursor lock status.

This method changes the cursor display mode using GLFW:

  • When cursor is locked (m_CursorLocked is true), the cursor is disabled and hidden
  • When cursor is unlocked, the cursor is set to normal mode and becomes visible
Note
Uses the native window handle obtained from the window object
Utilizes GLFW's input mode settings to control cursor visibility and behavior
